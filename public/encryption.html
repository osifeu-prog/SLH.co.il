למה זה קורה בכלל?

כשאפליקציה ישנה שומרת טקסט עברי בקידוד Windows-1255 או ISO-8859-8, אבל המשתמש מציג אותו ב-UTF-8 — כל תו עברי מתפצל ל-2-3 תווים של Latin Extended. הנה מה שקורה למילה "שלום":

איך SLH Spark מטפלת בעברית מהיסוד

בניגוד למערכות legacy, כל אחד מ-225+ ה-API endpoints שלנו משתמש ב-UTF-8 מהרגע הראשון:

  • DB: PostgreSQL 15 עם ENCODING UTF8 + LC_COLLATE 'he_IL.UTF-8' — מיון נכון של אלף-בית
  • API: FastAPI עם ensure_ascii=False בכל JSON response — עברית נשארת עברית
  • Frontend: כל HTML עם <meta charset="UTF-8"> + dir="rtl" + lang="he"
  • Bots: aiogram 3.x עם Python 3.11 — UTF-8 native, בלי escape hacks

למה זה חשוב למשקיעים?

בפרויקט crypto ישראלי, אם הכתובית על כפתור "לחץ לתשלום" נראית ×œ×—×¥ ×œ×ª×©×œ×•×, אנשים לא ילחצו. UX שבור = אמון שבור = כסף שלא זז. לכן אנחנו בנינו את הכל מחדש — לא "תרגמנו" מאנגלית.

מה עם ההצפנה של הכסף עצמו?

זה סיפור אחר לגמרי. SLH משתמש ב:

  • BSC (Binance Smart Chain): BEP-20 חוזה חתום, audit-ready
  • TON Network: הצפנת secp256k1, integration עם toncenter
  • Internal tokens (AIC, ZVK, REP, ZUZ): append-only ledger עם hash chain
  • Admin auth: JWT + X-Admin-Key dual-factor, bcrypt password hashing
  • .env never in git: secrets רק ב-Railway variables
