# SLH Device Demo Integration Plan

## Goal
Connect the ESP device project to the public demo page so users can experience the device flow during development.

## Phases
1. Device boot/status mirroring to web demo
2. Simulated WiFi onboarding in browser
3. Wallet status mock + real API bridge
4. Transaction approval demo
5. Device + web synchronized preview

## Core idea
Every firmware milestone should have:
- Physical device behavior
- Browser demo mirror
- Short release note
- Task/report update

## Immediate next steps
- Stabilize device firmware boot
- Add visual light mapping mode
- Add larger typography
- Add status packet format for future API/demo sync
